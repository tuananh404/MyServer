Tôi muốn xây dựng một hệ thống Web Panel Dashboard (chỉ dành cho Owner/Admin) chạy trên nền tảng Vercel, kết nối với Database Supabase (PostgreSQL). Mục đích của Web Panel này là quản lý, tạo Key/Token và cấu hình API. Sau khi tạo Key trên Web, phía client UI ImGui (C++) sẽ sử dụng trực tiếp Link Web này để gửi request Login Key và xác thực thiết bị (Device ID/Hardware ID).

Hãy đóng vai một chuyên gia Full-stack viết toàn bộ mã nguồn theo cấu trúc dự án hoàn chỉnh gồm: Database SQL, Backend API (Node.js/Express trên Vercel), và Frontend UI Dashboard (Dùng HTML/CSS/JS thuần hoặc React/Next.js tùy bạn chọn để chạy trên Vercel).

---

### PHẦN 1: CẤU TRÚC DATABASE (SUPABASE SQL)
Hãy cung cấp đoạn mã SQL để tạo các bảng sau:
1. Bảng `keys_management`:
   - `id` (int, primary key, tự tăng)
   - `key_string` (text, unique) - Chuỗi Key/Token ngẫu nhiên cấp cho user.
   - `duration_days` (int) - Thời hạn (1, 3, 7, 30, 90 ngày hoặc -1 nếu là Custom).
   - `expires_at` (timestamp, nullable) - Hết hạn (tính từ lúc user login lần đầu).
   - `hwid` (text, nullable) - Hardware ID / Device ID của máy chạy ImGui khi kích hoạt.
   - `status` (text) - Trạng thái ('unactivated', 'activated', 'banned', 'expired').
   - `note` (text) - Ghi chú tên khách hàng hoặc thông tin thêm.
   - `created_at` (timestamp, default now())

2. Bảng `fraud_logs`:
   - `id` (int, primary key, tự tăng)
   - `hwid` (text) - HWID của kẻ gian lận.
   - `reason` (text) - Lý do (Ví dụ: "Bypass Login", "Sai Key").
   - `logged_at` (timestamp, default now())

---

### PHẦN 2: NODE.JS BACKEND API (DEPLOY VERCEL)
Viết các API kết nối Supabase, xử lý đầy đủ CORS (để ImGui gọi được qua cURL/httplib) và bảo mật:

A. API Quản lý dành cho Web Dashboard (Cần check Header `Authorization: Bearer ADMIN_PASSWORD`):
1. [POST] `/api/admin/create-key` -> Body: { duration_days, note }. Sinh chuỗi ngẫu nhiên dạng `KEY_XXXXXX`, lưu status='unactivated'.
2. [POST] `/api/admin/reset-hwid` -> Body: { key_string }. Đặt `hwid = null`, status='unactivated' để user đổi máy.
3. [DELETE] `/api/admin/delete-key` -> Body: { key_string }. Xóa key.
4. [GET] `/api/admin/get-keys` -> Lấy danh sách toàn bộ key.
5. [GET] `/api/admin/get-fraud` -> Lấy danh sách nhật ký gian lận (Hiển thị Note Đỏ).

B. API Xác thực dành cho UI ImGui (Client Gọi):
1. [POST] `/api/client/login`
   - Body nhận từ ImGui: { key_string: string, hwid: string }
   - Logic xử lý:
     - Nếu Key không tồn tại: Ghi log vào `fraud_logs` với lý do "Sai Key", trả về `{ success: false, message: "Invalid Key" }`.
     - Nếu Key hợp lệ & 'unactivated': Cập nhật `hwid` của máy đó vào DB, chuyển status='activated', tính toán `expires_at = hiện tại + số ngày`. Trả về `{ success: true, token: "JWT_TOKEN_HOAC_SESSION", expires_at: ... }`.
     - Nếu Key đã 'activated': Kiểm tra `hwid` gửi lên có khớp trong DB không. Nếu khớp và chưa hết hạn -> Trả về `success: true`. Nếu sai hwid -> Trả về lỗi "Key đã dùng cho máy khác".
     - Nếu hết hạn -> Đổi status='expired', trả về lỗi.

### PHẦN 3: FRONTEND OWNER DASHBOARD WEB PANEL
Hãy viết giao diện Trang Web quản lý trực quan, bảo mật và scannable công việc tại một màn hình duy nhất:
1. Màn hình Đăng nhập (Login): Yêu cầu nhập Admin Password (lấy từ biến môi trường `.env`). Nếu đúng mới cho vào Dashboard.
2. Giao diện Dashboard chính:
   - Khu vực Tạo Key: Có các nút nhanh (1 ngày, 3 ngày, 7 ngày, 30 ngày, 90 ngày) và một ô nhập số ngày Custom kèm ô nhập Ghi chú (Note). Bấm nút sẽ gọi API tạo key và hiển thị ngay key vừa tạo để copy.
   - Khu vực Quản lý (Bảng danh sách Key): Hiển thị danh sách Key, Thời gian hết hạn, HWID đang kết nối, Trạng thái. Mỗi dòng có 2 nút chức năng: **[Reset HWID]** và **[Xóa Key]**.
   - Khu vực Cảnh báo (Note đỏ gian lận): Một bảng riêng hiển thị danh sách từ bảng `fraud_logs`. Nếu có dữ liệu mới, vùng này sẽ đổi thành **Màu Đỏ Cảnh Báo** nhấp nháy hiển thị chi tiết HWID vi phạm để Owner dễ dàng nhận biết.

Hãy cung cấp toàn bộ mã nguồn chi tiết, cấu trúc file rõ ràng và hướng dẫn điền file `.env` (gồm URL Supabase, Anon Key, và Admin Password) để tôi deploy lên Vercel chỉ trong một cú click.
